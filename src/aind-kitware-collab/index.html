Additional Notes

  • Input source data is obtained as follows:
    1. Data is acquired via the SmartSPIM light sheet microscope as TIFF stacks. Acquisition metadata is composed in an `acquisition.json` file.
    2. Data is stitched via TeraStitcher, converted to OME-Zarr format, and hosted on AWS S3.
  • Unless specified otherwise, the following defaults are employed:
    • SmartSPIM intensity images are selected at Level 4 in scale space, meaning each axis has been 2x downsampled four times. The resulting image is (1 / 2 ^ 4) the size of the acquisition volume along each axis or (1 / 2 ^ (3 * 4)) the size of the acquisition volume in total voxel volume.
    • Common Coordinate Framework v3.1 images are selected with 25um spacing, nonunit direction, and unit origin.
    • The registration pipeline executes as follows:
      1. SmartSPIM intensity image is overlaid on CCF image via resampling or origin update in ITK
      2. SmartSPIM intensity image is mapped to CCF image space via ITKElastix optimization of several transform stages:
        1. Rigid (rotation and translation only)
        2. Affine (rotation, translation, scale, skew)
        3. B-Spline deformations with control points at 500 micron spacings
  • SmartSPIM mesh geometries are generated for surface rendering in ITK-VTK-Viewer via the following process:
    1. Registration yields a transform from SmartSPIM to CCF space.
    2. We apply the registration transform to resample a composited expert label image from SmartSPIM to CCF space.
    3. To get a binary mesh without scalar data, we convert the new label image to a binary image and apply ITK::BinaryMask3DMeshSource to obtain a mesh geometry.
    4. To get surface metrics, we do the following for each label value:
      1. Get a binary image for the given label value from the CCF label atlas and use ITK::ApproximateSignedDistanceMapImageFilter to create a corresponding signed distance image.
      2. Create a binary image for the given label value from the resampled SmartSPIM label image and apply ITK::BinaryMask3DMeshSource to obtain a mesh geometry for the label region.
      3. Probe the CCF label signed distance image with the transformed SmartSPIM label mesh by visiting each mesh vertex and sampling the distance value at that point in physical space.
      4. Attach the probed distance value at each vertex to the label mesh as scalar data and save to disk.
      5. Evalute registration alignment by considering the distribution of signed distances over the 3D label mesh surface with 1D statistics.
